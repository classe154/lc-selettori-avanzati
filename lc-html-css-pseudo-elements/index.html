Template per Pseudo-elementi CSS

Gli pseudo-elementi permettono di stilizzare parti specifiche di un elemento o aggiungere contenuto generato dal CSS.

1. Task List con Pseudo-elementi e Centralizzazione

Esempio di come centralizzare le regole generali e differenziare solo il necessario con classi specifiche.

Risultato Visivo:

  • Studiare HTML
  • Imparare CSS base
  • Completare il progetto entro ieri
  • Studiare JavaScript
  • Configurare l'ambiente di sviluppo
  • Fare tutto in un giorno
  • Creare il portfolio

Codice CSS (Principio di Centralizzazione):

.task-list li::before {
content: ""; /* Verrà sovrascritto */
position: absolute;
left: 12px;
top: 50%;
transform: translateY(-50%);
font-size: 16px;
font-weight: bold;
}

/* Classi SPECIFICHE per differenziare solo content e colore */
.task-list li.completed::before {
content: "\2713"; /* Entità CSS checkmark */
color: #28a745; /* Verde */
}

.task-list li.failed::before {
content: "\2717"; /* Entità CSS X */
color: #dc3545; /* Rosso */
}

Principio di Centralizzazione: Tutte le regole comuni (posizionamento, dimensioni, font) sono definite una sola volta nell'elemento base. Le classi specifiche (.completed, .failed,) sovrascrivono solo le proprietà che devono essere diverse (content e color).

2. Dove Appaiono ::before e ::after

Concetto chiave: Gli pseudo-elementi non appaiono "prima" o "dopo" l'elemento, ma prima e dopo il suo contenuto!

Visualizzazione Pratica:

QUESTO È IL CONTENUTO DELL'ELEMENTO
Osserva: ::before appare prima del testo "QUESTO È IL CONTENUTO..." e ::after appare dopo, ma entrambi sono comunque dentro l'elemento!

Flow del Contenuto:

Questo è il contenuto reale dell'elemento HTML
.content-flow::before {
content: "BEFORE (prima del contenuto)";
/* Appare PRIMA del testo, ma DENTRO l'elemento */
}

.content-flow::after {
content: "AFTER (dopo il contenuto)";
/* Appare DOPO il testo, ma DENTRO l'elemento */
}

3. Esempio Avanzato: Lista Numerata Custom

Uso di counter() con pseudo-elementi per numerazione personalizzata.

Risultato:

  1. Primo elemento della lista
  2. Secondo elemento con numerazione custom
  3. Terzo elemento con stile personalizzato
  4. Quarto elemento per vedere la progressione

Codice CSS:

.numbered-list {
list-style: none;
counter-reset: item-counter; /* Inizializza counter */
padding-left: 0;
}

.numbered-list li {
counter-increment: item-counter; /* Incrementa ad ogni li */
padding: 10px 15px 10px 50px;
position: relative;
}

.numbered-list li::before {
content: counter(item-counter); /* Mostra il numero */
position: absolute;
left: 15px;
background: #4285f4;
color: white;
width: 25px;
height: 25px;
border-radius: 50%;
display: flex;
align-items: center;
justify-content: center;
}

4. Regole Fondamentali per gli Pseudo-elementi

📋 Checklist Essenziale:

  • content è obbligatorio: Senza content: ""; lo pseudo-elemento non appare!
  • Due volte due punti: ::before e ::after (non :before)
  • Sono elementi inline di default: Usa display: block se serve
  • Non funzionano su elementi void: No su <img>, <input>, <hr>
  • Sono figli dell'elemento: Non fratelli, ma contenuti dentro
  • Accessibilità: Il contenuto generato non è letto dai screen reader

Entità CSS Utili:

Ecco alcuni codici Unicode utili per gli pseudo-elementi:

  • content: "\2713"; - ✓ Checkmark
  • content: "\2717"; - ✗ X mark
  • content: "\2605"; - ★ Stella piena
  • content: "\2606"; - ☆ Stella vuota
  • content: "\25B6"; - ▶ Freccia
  • content: "\26A0"; - ⚠ Warning
  • content: "\2139"; - ℹ Info
  • content: "\23F3"; - ⏳ Pending/Loading
